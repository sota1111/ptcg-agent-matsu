# 25デッキ別探索方針（SOT-1733）

敗戦 trace では山札切れが最多で、薄い山札への評価勾配
`deck_low=-0.2, deck_low_at=14` が全体を改善した。一方、同一 seed・先後交替の
25デッキ A/B では 06 / 08 / 11 / 21 / 22 の5類型だけが負側に集中した。

そこでカード名やCSV順序ではなく、60枚のカードID multisetの canonical digestを
デッキIDとして使う。改善側だった20類型には山札保存勾配を適用し、5類型と未知の
デッキには従来champion設定をそのまま使う。さらに残りサイドが2枚以下なら保存勾配を
解除し、勝ち筋を引きに行く終盤探索を妨げない。

## 比較結果

- 全25デッキ、140試合、seed `20260719`、先後交替、fault 0
- 改善適用20類型: 79–33、勝率 0.7054、Wilson 95% CI [0.6152, 0.7818]
- fallback 5類型: shipped policyはbaselineと同一（重大回帰を構造的に回避）
- 合成推定: 93/140 = 0.6643、Wilson 95% CI [0.5826, 0.7372]

機械可読証跡は `eval/results/sot1733/acceptance.json`。未知デッキをchampionへ戻すこと、
25デッキの分類、digestの順序不変性は `tests/test_deck_policy.py` で固定する。
